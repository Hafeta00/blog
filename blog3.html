Flexbox (and grid) are ways to describe the layout of your user interface. The parts of a webpage are split up into many boxes. Previously, all you could do with the boxes was play with: dimensions (width, height, margins, etc), positioning (normal, fixed, fixed even when scrolling, etc), alignment (left, right, center, top, bottom), and which boxes were or weren't allowed to sit next to which other boxes (floating). Sometimes you couldn't even do all those things. For example vertical centering has always been unneccesarily complex. These limitations meant arbitrary layouts were difficult or impossible to pull off. It was satisfactory for documents, but not for complex user interfaces.

This is an example of flexbox stretching only the middle element, and having both the right and left elements be a fixed size. We can set a fixed width for elements that should remain static, and set flexgrow: 1 for elements that should be stretched:


.container {
display: flex;
}
.item {
height: 100px;
width: 100px; /* A fixed width as the default */
}
.item-center {
flex-grow: 1; /* Set the middle element to grow and stretch */
}
.item + .item {
margin-left: 2%;
}

Instead of using floats or inline-block, you can use flexbox. It gives you a ton of additional options; make the items grow so they're all the same height; make them butt up against the outside edges of the parent container but put space between themselves; order the items one way for desktop, and another for smaller devices. There's tons of things flexbox can do. It's pretty well supported too, to have it work in all current major browsers, you just need the webkit prefix.
